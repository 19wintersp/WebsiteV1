About me

Hello! I'm a human who likes to tell computers what to do. My first "proper" programming language was Python, however I mostly use NodeJS, Web Technologies and C++ now. I'll also occasionally use Java and C#, though the the JVM and .NET Framework annoy me a little. I also want to use Deno, but Replit is being a bit annoying at the moment by using a version that's horribly out of date.

At the moment, I'm working on a few projects, which you can see more of in Projects. My main foci are a stock-trading simulation game, a music notation app, and a fully-featured voice- and text- chat application; all of which are built with Web Technologies.

Projects

Here's a list of some of the projects I've made or am making. You can see all of them on my Replit profile.

Polyphony (Preview)

Site | Code | Post

Polyphony Preview is a site used to test technologies which will eventually become a part of Polyphony: my voice- and text- chat application which is essentially just a cheap knockoff of Discord. PolyphonyPreview lets you chat through messages and group calls, along with private, shareable rooms. Don't worry though, the finished version will look much better, have many more features, and actual accounts and permanent messages.

ConnectFour Bot

Code | Post

My ConnectFour bot was a test of C++, as it was one of the first relatively complex programs I made in it. The bot used "reality branches" to compute its best move, however it struggled on the speed side. I'm still trying to figure out how to use multithreading. I suggest that you fork this repl and fiddle around with some of the configuration, and see if you can tune-out some of the bugs it has.

GameMaker GUI

Site | Code | Post

GameMaker GUI was the web version of my unsuccessful GameMaker; both of which were designed to make it easier to create choose-your-own-adventure-style games. They both make use of a system to serialize (non-binary) trees, albeit with one much less exciting than the other. It's a bit hard to use, and the Help function is a bit useless, but it does work relatively well. It was also the prototype of my "new" design language (which should probably be named "Bland and Ugly") which was influenced by Material Design and Google, though with some elements from Metro.

XHTML Introduction*

Site | Code | Post

An Introduction to XHTML. Made by @xxpertHacker, this webpage introduces you to the magical world of XML being used to create webpages. I strongly suggest you have a look at it. I played a minor role in doing some CSS stuff and proofreading the draft a bit.
